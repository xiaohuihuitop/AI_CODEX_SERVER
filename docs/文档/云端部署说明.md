# Codex 云端中继部署说明

## 架构

```text
手机浏览器 -> 云端 relay -> Windows desktop-agent -> 本机 Codex Desktop
```

云端只负责 HTTPS/WSS 中继和 token 鉴权。Codex Desktop 的 `127.0.0.1:9229` 仍然只在 Windows 本机访问，不暴露到公网。

## 云端资源

- 个人使用：1 vCPU、1 GB 内存、20 GB SSD。
- 只需要开放 `80/443`。
- 不需要 GPU。

## Docker 启动

复制环境变量模板：

```powershell
Copy-Item .env.example .env
```

编辑 `.env`：

```text
CODEX_CLOUD_TOKENS=codex_home_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

启动：

```powershell
docker compose up -d --build
```

## GitHub Actions 镜像产物

推送 `build-*` 标签会自动执行 GitHub Actions，产出 Docker 镜像 tar 包 artifact：

```powershell
git tag build-v1.0
git push origin build-v1.0
```

工作流会执行：

```text
npm ci
npm test
npm run check
docker build
docker save
```

产物名称类似：

```text
ai-codex-server-build-v1.0.tar
```

服务器下载后导入：

```powershell
docker load -i ai-codex-server-build-v1.0.tar
```

如果有正式域名，把 `Caddyfile` 第一行 `:80` 改成你的域名：

```caddyfile
codex.example.com {
  reverse_proxy relay:8787
}
```

Caddy 会自动申请 HTTPS 证书。

## Windows 电脑端 Agent

在 Windows 电脑上运行：

```powershell
$env:CODEX_CLOUD_URL="https://codex.example.com"
$env:CODEX_DEVICE_TOKEN="codex_home_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
$env:CODEX_DEVICE_NAME="home-pc"
npm run start:agent
```

手机访问：

```text
https://codex.example.com/?token=codex_home_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

## 桌面管理小软件雏形

当前提供本地管理页面：

```powershell
npm run start:manager
```

打开：

```text
http://127.0.0.1:8790
```

它可以管理云端地址、固定 token、设备名，并生成手机访问地址和 Agent 环境变量。后续可以用 Electron 包成真正的托盘小软件。

## 多电脑

一台电脑一个 token：

```text
codex_home_xxx   -> 家里电脑
codex_office_xxx -> 办公电脑
```

云端 `CODEX_CLOUD_TOKENS` 用逗号分隔：

```text
CODEX_CLOUD_TOKENS=codex_home_xxx,codex_office_xxx
```

相同 token 同时只能有一个 Agent 在线，避免误控。

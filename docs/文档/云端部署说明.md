# Codex 云端中继部署说明

## 架构

```text
手机浏览器 -> 云端 relay -> Windows desktop-agent -> 本机 Codex Desktop
```

云端只负责 HTTP/WS 或 HTTPS/WSS 中继和 token 鉴权。Codex Desktop 的 `127.0.0.1:9229` 仍然只在 Windows 本机访问，不暴露到公网。
当前默认部署方式为 HTTP 直连：手机和 Windows Agent 通过 `http://群晖地址:8008` 访问云端 relay。

## 云端资源

- 个人使用：1 vCPU、1 GB 内存、20 GB SSD。
- 默认只需要开放 `8008`。
- 不需要 GPU。

## 群晖部署路径

建议把部署文件放到：

```text
/volume1/SSD/docker/codex_server
```

目录中需要保留：

```text
docker-compose.yml
.env
ai-codex-server-build-v1.0.tar
```

## Docker 启动

进入部署目录：

```sh
mkdir -p /volume1/SSD/docker/codex_server
cd /volume1/SSD/docker/codex_server
```

从 GitHub Release 下载：

```sh
wget https://github.com/xiaohuihuitop/AI_CODEX_SERVER/releases/download/build-v1.0/ai-codex-server-build-v1.0.tar
wget https://github.com/xiaohuihuitop/AI_CODEX_SERVER/releases/download/build-v1.0/docker-compose.yml
wget https://github.com/xiaohuihuitop/AI_CODEX_SERVER/releases/download/build-v1.0/.env.example -O .env
```

编辑 `.env`：

```text
CODEX_CLOUD_TOKENS=codex_home_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
CODEX_CLOUD_HTTP_PORT=8008
```

导入镜像：

```sh
docker load -i ai-codex-server-build-v1.0.tar
```

启动：

```sh
docker compose up -d
```

验证：

```sh
curl "http://127.0.0.1:8008/codex/health?token=codex_home_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
```

手机访问：

```text
http://群晖IP:8008/?token=codex_home_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

## GitHub Release 镜像产物

推送 `build-*` 标签会自动执行 GitHub Actions，产出 Docker 镜像 tar 包，并上传到对应 GitHub Release：

```powershell
git tag build-v1.0
git push origin build-v1.0
```

工作流会执行：

```text
npm ci
npm test
npm run check
docker buildx build
upload artifact
publish release asset
```

Releases 页面附件名称类似：

```text
ai-codex-server-build-v1.0.tar
docker-compose.yml
.env.example
```

服务器下载后导入：

```sh
docker load -i ai-codex-server-build-v1.0.tar
```

## 可选 HTTPS

默认不启用 HTTPS，也不占用 `80/443`。

如果后续要接入正式域名和 HTTPS，可以在群晖反向代理、Nginx、Caddy 或其他网关上把外部 HTTPS 转发到：

```text
http://127.0.0.1:8008
```

## Windows 电脑端 Agent

在 Windows 电脑上运行：

```powershell
$env:CODEX_CLOUD_URL="http://群晖IP:8008"
$env:CODEX_DEVICE_TOKEN="codex_home_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
$env:CODEX_DEVICE_NAME="home-pc"
npm run start:agent
```

手机访问：

```text
http://群晖IP:8008/?token=codex_home_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

## 桌面管理小软件雏形

当前提供本地管理页面：

```powershell
npm run start:manager
```

打开：

```text
http://127.0.0.1:8790
```

它可以管理云端地址、固定 token、设备名，并生成手机访问地址和 Agent 环境变量。后续可以用 Electron 包成真正的托盘小软件。

## 多电脑

一台电脑一个 token：

```text
codex_home_xxx   -> 家里电脑
codex_office_xxx -> 办公电脑
```

云端 `CODEX_CLOUD_TOKENS` 用逗号分隔：

```text
CODEX_CLOUD_TOKENS=codex_home_xxx,codex_office_xxx
```

相同 token 同时只能有一个 Agent 在线，避免误控。

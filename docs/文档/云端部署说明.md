# Codex 云端中继部署说明

## 架构

```text
手机浏览器 -> 云端 relay -> Windows desktop-agent -> 本机 Codex Desktop
```

云端只负责 HTTP/WS 或 HTTPS/WSS 中继和 token 鉴权。Codex Desktop 的 `127.0.0.1:9229` 仍然只在 Windows 本机访问，不暴露到公网。
当前默认部署方式为 HTTP 直连：手机和 Windows Agent 通过 `http://群晖地址:8008` 访问云端 relay。

## 三端目录

当前仓库按端拆分：

```text
server/          云端 relay 和 Docker 部署文件
desktop-client/  Windows 桌面管理器和 Agent
mobile-app/      Android uni-app 手机端
```

云端 Docker 构建上下文固定为 `server/`。服务端不依赖 `desktop-client/` 或 `mobile-app/` 目录。

## 云端资源

- 个人使用：1 vCPU、1 GB 内存、20 GB SSD。
- 默认只需要开放 `8008`。
- 不需要 GPU。

## 群晖部署路径

建议把部署文件放到：

```text
/volume1/SSD/docker/codex_server
```

目录中需要保留：

```text
docker-compose.yml
.env

```

## Docker 启动

进入部署目录：

```sh
mkdir -p /volume1/SSD/docker/codex_server
cd /volume1/SSD/docker/codex_server
```

从 GitHub Release 下载：

```sh
wget -O ai-codex-server-build-v1.7.tar https://github.com/xiaohuihuitop/AI_CODEX_SERVER/releases/download/build-v1.7/ai-codex-server-build-v1.7.tar
wget -O docker-compose.yml https://github.com/xiaohuihuitop/AI_CODEX_SERVER/releases/download/build-v1.7/docker-compose.yml
wget -O .env https://github.com/xiaohuihuitop/AI_CODEX_SERVER/releases/download/build-v1.7/env.example
```

编辑 `.env`：

```text
CODEX_CLOUD_TOKENS=codex_home_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
CODEX_CLOUD_HTTP_PORT=8008
```

导入镜像：

```sh
sudo docker load -i ai-codex-server-build-v1.7.tar
```

启动：

```sh
sudo docker-compose config
sudo docker-compose up -d
```

验证：

```sh
curl "http://127.0.0.1:8008/codex/health?token=codex_home_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
```

手机访问：

```text
http://群晖IP:8008/?token=codex_home_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

## 群晖常见问题

启动前先确认部署目录和 `.env`：

```sh
cd /volume1/SSD/docker/codex_server
pwd
ls -la
cat .env
sudo docker-compose config
```

`.env` 至少需要包含：

```text
CODEX_CLOUD_TOKENS=codex_home_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
CODEX_CLOUD_HTTP_PORT=8008
```

如果看到 `CODEX_CLOUD_TOKENS variable is not set`，说明当前目录没有正确的 `.env`，或者 `.env` 里没有填写 `CODEX_CLOUD_TOKENS`。

如果看到 `cpus expected type 'float32'` 或 `NanoCPUs can not be set`，说明使用了旧版本的 `docker-compose.yml`，需要重新下载 `build-v1.7` 的 `docker-compose.yml`。

当前发布的群晖 compose 不设置 `cpus`、`mem_limit` 等资源限制，避免依赖 NAS 内核的 CPU CFS/cgroup 能力。个人使用时云端 relay 很轻量，如需限制资源，建议在群晖 Docker 管理界面或系统层配置。

## 可选 HTTPS

默认不启用 HTTPS，也不占用 `80/443`。

如果后续要接入正式域名和 HTTPS，可以在群晖反向代理、Nginx、Caddy 或其他网关上把外部 HTTPS 转发到：

```text
http://127.0.0.1:8008
```

## Windows 电脑端 Agent

在 Windows 电脑上运行：

```powershell
cd desktop-client
npm install
$env:CODEX_CLOUD_URL="http://群晖IP:8008"
$env:CODEX_DEVICE_TOKEN="codex_home_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx"
$env:CODEX_DEVICE_NAME="home-pc"
npm run start:agent
```

手机访问：

```text
http://群晖IP:8008/?token=codex_home_xxxxxxxxxxxxxxxxxxxxxxxxxxxxxxxx
```

## Windows GUI 管理器

启动 Windows 图形管理器：

```powershell
cd desktop-client
npm install
npm run start:manager:gui
```

它会打开一个 Windows 窗口，可以保存云端地址、固定 token、设备名，生成手机访问地址和 Agent 环境变量，并在窗口里启动或停止 Windows Agent。
窗口会显示三个状态：

- 云端 relay 是否可访问，以及云端是否显示 Agent 在线。
- Windows Agent 是否由管理器启动并运行。
- Codex Desktop 的本机 CDP 调试端口 `127.0.0.1:9229` 是否可用。

注意：GUI 不提供其他控制路径。如果 Codex Desktop 没有开放 `127.0.0.1:9229`，手机端可以连到云端和 Agent，但不能读取线程或发送消息。

如需调试，也保留本地 Web 管理入口：

```powershell
cd desktop-client
npm run start:manager
```

## 多电脑

一台电脑一个 token：

```text
codex_home_xxx   -> 家里电脑
codex_office_xxx -> 办公电脑
```

云端 `CODEX_CLOUD_TOKENS` 用逗号分隔：

```text
CODEX_CLOUD_TOKENS=codex_home_xxx,codex_office_xxx
```

相同 token 同时只能有一个 Agent 在线，避免误控。

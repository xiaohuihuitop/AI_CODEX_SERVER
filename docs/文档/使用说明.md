# Codex Windows Bridge 使用说明

## 目标

在 Windows 上提供一个局域网 Web 控制端，让手机浏览器可以查看 Codex Desktop 线程、读取历史消息，并把输入发送到电脑端 Codex Desktop。
页面使用两级选择：先选择 Codex Desktop 中打开的项目文件夹，再选择该项目下打开的对话。

## 实现方式

- 本机服务通过 Codex Desktop 的本机 CDP 调试端口读取当前打开的线程，再用 `%USERPROFILE%\.codex\sessions` 和 `%USERPROFILE%\.codex\session_index.jsonl` 匹配 threadId 和历史消息。
- 同一个项目文件夹下可以显示多个当前打开对话。
- 手机端页面通过 HTTP API 调用本机服务。
- 手机端消息使用本地 Markdown 渲染器展示标题、列表、引用、代码块、链接和表格。
- 手机端使用灰色或绿色状态点显示当前选择对话是否正在工作。
- 发送消息时，服务使用 Windows PowerShell 脚本连接 Codex Desktop 的本机 CDP 调试端口 `127.0.0.1:9229`。
- 控制脚本在 Codex Desktop 界面中定位目标项目和线程，向 `.ProseMirror` 输入框插入文本，并点击发送按钮。

## 前置条件

- Windows 环境。
- Node.js 20 或更高版本。
- Codex Desktop 正在运行。
- Codex Desktop 已开放本机 CDP 调试端口 `127.0.0.1:9229`。
- 手机和电脑处在同一个局域网。

## 启动

### Windows 桌面管理器

正式桌面入口使用 Electron 小工具。在项目目录执行：

```powershell
npm run start:manager:gui
```

窗口名称为 `Codex Desktop 管理器`，用于配置云端服务器地址、固定 Token、设备名称，并启动或停止 Windows Agent。
当前 PowerShell WinForms 入口仅作为旧版备用入口：

```powershell
npm run start:manager:gui:legacy
```

### 本机局域网服务

在项目目录执行：

```powershell
npm start
```

服务默认监听 `0.0.0.0:8787`，启动日志会输出电脑本机地址和局域网地址，例如：

```text
http://localhost:8787/?token=dev-token
http://192.168.1.31:8787/?token=dev-token
```

电脑浏览器使用 `localhost` 地址，手机浏览器使用局域网 IP 地址。

## API

- `GET /codex/health`：服务健康检查。
- `GET /codex/threads`：读取 Codex Desktop 当前打开的线程列表。
- `GET /codex/history?thread=<id>`：读取指定线程历史。
- `GET /codex/status?thread=<id>&since=<iso-time>`：读取指定时间之后的回复状态。
- `POST /send`：发送消息到指定 Codex Desktop 线程。
- `POST /codex/stop`：向 Codex Desktop 发送停止回复指令。

## 验证

```powershell
npm test
npm run check
powershell.exe -NoProfile -ExecutionPolicy Bypass -Command "$null = [scriptblock]::Create((Get-Content -LiteralPath '.\scripts\win-codex-control.ps1' -Raw)); 'ok'"
```

## 注意事项

- 当前正式控制链路只使用 CDP，不使用 `codex://threads/<id>`。
- 如果 Codex Desktop 没有开放 `127.0.0.1:9229`，发送控制会直接失败并返回错误。
- 不包含自动兜底控制路径；新增其他控制策略前需要先确认。
